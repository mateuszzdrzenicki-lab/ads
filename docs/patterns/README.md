# Patterns

Patterns are multi-component UX recipes — how components combine to solve a recurring
job: forms, empty states, destructive confirmation, page headers, and the like. A pattern
is only worth writing once there are several components to compose.

## Nothing here yet

**No patterns exist.** The system currently has one component (Button), so there is nothing
to compose. This file is a placeholder so the empty directory isn't mistaken for missing
docs — don't infer a pattern from an absence, and don't author one speculatively.

When a second component lands and a real recurring composition appears, add one file per
pattern here. Until then, see `../constraints.md` for what exists and the stop-and-report
rule at the edge of the system.
